Everything Sports

Fantasy basketball tools to help the saavy fantasy player!

This was a web application I made for my final project in my Web Development class. There were no strict criteria, just to apply the things we learned in class, and use them in our project. So I thought to myself, what is something I think would be useful for not only myself, but others as well, and I thought of my fantasy basketball team. The website I play on doesn’t have the best features, and doesn’t even allow you to sort your team based on statistics, so right there I saw a need for something to help users more easily be able to look at their team.

So I first thing I needed to do was to figure out what I was going to need to accomplish this. I came to the conclusion that I would need a way to obtain player statistics because this tool is meant for users to be able to sort their team by points or assists for example. So the solution I came to was to scrape information from basketball-reference.com, and I did this by leverage my knowledge of Python since that seemed like the optimal way to obtain information from a web page.

So how I did this specifically was by utilizing the Beautiful Soup library, which provides methods to help parse a web page’s html. I had to first analyze the html of the page that I was scraping and understand the structure of the page. Once I understood the structure, then I used Beautiful Soup to search for certain tags and be able to extract the specific information I am looking for. Some problems I ran into were some special cases such as a field being blank which I had to replace with “0”, also I had to change the encoding of the text to ASCII for it to store correctly.

Once the page was parsed, I put each individual player’s statistics in a dictionary with keys being the statistic name (i.e. “FG% or Pts”), and I placed that within a dictionary under the key “stats”, and placed another key/value pair, representing their name. The new dictionary is then placed within a list which is then outputted as a JSON file.

Then I used JavaScript to parse any text a user submits and split that text by whitespace using a RegEx. I then checked each combination of two words (a name) to see if that name was a value in my playerDictionary, and if it was, then that is a player.

The front-end is built on Angular, and some things I used it for are to display the table which contains the players using ng-repeat, ng-hover to highlight columns, sortBy for a custom sorting function. Also used to populate various dropdowns, especially the dropdowns containing all possible players. Ng-hide, ng-class, ng-clicks used throughout as well to provide functionality throughout the site.

The player comparison feature came last, and was created using D3.js. I wanted to be able to visually compare players, and so I thought that using D3 would be a good way to do this. I implemented the ability to compare as many players as wanted, as well as choice of category and grouping preference.
